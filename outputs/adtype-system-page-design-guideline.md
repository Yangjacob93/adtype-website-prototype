# Adtype 기술/시스템 상세 페이지 디자인 가이드

기준 페이지: `adtype-system-market-consumer-analysis.html`  
적용 범위: Adtype의 기술, 시스템, 프로덕트 기반 서비스 상세 페이지

이 문서는 `시장·소비자 분석` 페이지를 만들며 확정한 디자인 시스템과 섹션 구성 규칙을 정리한 가이드입니다. 이후 `캠페인 과제 도출`, `공간 분석·미디어플래닝`, `크리에이티브 제작`, `캠페인 집행·운영`, `성과·인사이트 분석` 같은 시스템 페이지를 만들 때 같은 기준으로 확장합니다.

## 1. 페이지 성격

이 페이지는 SaaS 제품을 단독 판매하는 페이지가 아닙니다.  
Adtype이 광고 서비스를 제공하는 과정에서 기술과 시스템을 어떻게 활용해 서비스 품질을 높이는지 보여주는 B2B 서비스 상세 페이지입니다.

따라서 페이지의 목적은 다음입니다.

- 제품 기능을 나열하는 것이 아니라, 광고 캠페인 품질이 어떻게 좋아지는지 설명한다.
- 고객의 일을 대신해 “전략을 세워준다”는 인상보다, 고객의 브랜드 성장 방향을 데이터로 확인하고 캠페인 과제를 도출한다는 인상을 준다.
- 기술을 과시하기보다, 기술이 실무 판단과 산출물로 어떻게 연결되는지 보여준다.
- B2B 마케터, 광고인, CEO가 읽었을 때 “이 회사는 감이 아니라 근거로 움직인다”는 확신을 주는 것이 핵심이다.

## 2. 전체 IA

기술/시스템 상세 페이지의 기본 구성은 다음 순서를 따른다.

1. Hero
2. Brand Alignment Intro
3. Problem / Solution Matrix
4. Product / System Detail
5. FAQ
6. Final CTA

### 2.1 Hero

역할: 이 페이지가 어떤 시스템 단계인지 즉시 선언한다.

구성:

- 상단 내비게이션
- 배경 영상 또는 현장성이 있는 이미지
- 작은 단계 번호
- 큰 페이지 타이틀
- 짧은 설명 카피

현재 기준:

- 단계 번호: `01.`
- 헤드라인: `시장·소비자 분석`
- 서브카피:  
  `브랜드의 성장 방향을 소비자 데이터로 확인하고,`  
  `OOH 캠페인이 집중해야 할 과제와 메시지 기회를 찾습니다.`

디자인 원칙:

- Hero는 첫 화면 전체를 꽉 채우지 않는다. 서브 페이지이므로 약 `80vh` 정도가 적절하다.
- 배경은 영상이 가장 좋다. 단, 텍스트 가독성을 위해 강한 딤을 적용한다.
- breadcrumb 성격의 `홈 > 시스템 > ...` 요소는 제거한다. 정보값보다 장식성이 크고, 첫인상을 복잡하게 만든다.
- 단계 번호는 작게 두되 너무 작아 보이지 않게 한다.
- 모바일에서는 `01.`을 타이틀 위의 작은 섹션 라벨처럼 분리하고, 헤드라인은 한 줄에 들어오도록 조정한다.

## 3. 컬러 시스템

### 3.1 핵심 컬러

```css
--ad: #eb4200;
--ink: #151515;
--ink-2: #343434;
--muted: #6f747b;
--faint: #a5abb3;
--line: #e2e2e2;
--paper: #f7f7f7;
--white: #ffffff;
```

### 3.2 사용 원칙

- `#EB4200`은 Adtype의 핵심 액션, 강조 키워드, 선택 상태에만 사용한다.
- 본문 컬러는 `#151515`, `#343434`, `#6f747b` 안에서 해결한다.
- 파란색, 보라색, 베이지, 크림색, 과도한 그라디언트는 사용하지 않는다.
- 섹션 배경은 최대한 중립적인 회색과 흰색으로 정리한다.
- 따뜻한 주황빛 회색은 페이지를 마일드하고 밋밋하게 만들 수 있으므로 피한다.

### 3.3 섹션 배경 기준

- 기본 페이지 배경: `#f7f7f7`
- 흰색 콘텐츠 섹션: `#ffffff`
- 카드 면 또는 시스템 카드: `#f6f6f6`
- FAQ 섹션: `#f9f9f9`
- 강조 CTA 섹션: `#151515`

## 4. 폰트 시스템

### 4.1 기본 폰트

Pretendard를 기본으로 사용한다.

```css
font-family: "Pretendard Variable", Pretendard, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
letter-spacing: 0;
word-break: keep-all;
```

### 4.2 웨이트 원칙

Adtype 페이지에서는 과한 Bold를 피하되, 섹션의 핵심 문장은 충분히 단단하게 보이도록 한다.  
초기 기준의 핵심은 "웨이트를 낮게 유지"가 아니라, 본문과 보조 문장은 가볍고 선명하게 두고 섹션 헤드라인과 핵심 소제목은 한 단계 더 힘 있게 잡는 것이다.

권장 웨이트:

- Hero H1: `560`에서 `600`
- 일반 Section H2: `540`에서 `580`
- Product/System 모듈 H3: `540`에서 `580`
- 인터랙티브 보드/시각화 모듈 H3: 필요할 때만 `600`에서 `620`
- 카드 제목: `540`에서 `560`
- FAQ 질문: `540`
- 라벨/번호/포인트: `560`에서 `580`
- 강조 문장: `520`에서 `560`
- CTA 제목: `540`
- CTA 버튼: `560`
- 본문: `400`
- 보조 설명/캡션: `400`에서 `450`

주의:

- `700` 이상은 원칙적으로 사용하지 않는다.
- 너무 얇은 헤드라인은 B2B 서비스의 신뢰감이 약해 보일 수 있다.
- 너무 두꺼운 헤드라인은 촌스럽고 무거워 보일 수 있으므로 일반 텍스트는 `540~580`을 기본 상한으로 본다.
- `600~620`은 KPI 보드, 매트릭스, 동적 브리프 시트처럼 시각적 모듈 안에서 명확한 위계가 필요할 때만 제한적으로 사용한다.
- FAQ/CTA는 페이지군의 마무리 모듈이므로 1페이지와 동일하게 제목과 질문은 `540`, 버튼은 `560`을 기준으로 한다.

### 4.3 주요 타입 스케일

Hero:

```css
h1 {
  font-size: clamp(42px, 4.5vw, 64px);
  line-height: 1.05;
  font-weight: 560;
}
```

섹션 헤드라인:

```css
font-size: clamp(30px, 2.7vw, 44px);
line-height: 1.22;
font-weight: 580 ~ 620;
```

Product/System 섹션 헤드라인:

```css
font-size: clamp(30px, 3vw, 46px);
line-height: 1.22;
font-weight: 580 ~ 620;
```

본문:

```css
font-size: 15px 또는 16px;
line-height: 1.68 ~ 1.82;
font-weight: 400;
```

카드 내부 상세 본문:

```css
font-size: 14px;
line-height: 1.68 ~ 1.7;
font-weight: 400;
```

## 6. 섹션별 디자인 가이드

## 6.1 Hero

### 목적

해당 시스템 페이지의 역할을 한 문장으로 각인한다.

### 구조

```html
<header class="hero">
  <video class="hero__bg"></video>
  <div class="hero__copy">
    <span class="hero__index">01.</span>
    <h1>시장·소비자 분석</h1>
    <p class="hero__subcopy">...</p>
  </div>
</header>
```

### 디자인 규칙

- 배경은 실제 시스템 화면, OOH 현장, 캠페인 분석 흐름이 느껴지는 영상/이미지를 사용한다.
- 텍스트 위에는 카드를 얹지 않는다.
- 배경은 충분히 어둡게 딤 처리한다.
- Hero 텍스트는 중앙 정렬이 기본이다.
- 모바일에서는 헤드라인이 한 줄로 들어오도록 크기를 조정한다.

### 금지

- breadcrumb 과다 노출
- 장식용 라인
- 의미 없는 영어 라벨
- 배경을 너무 잘 보이게 해서 텍스트가 묻히는 구성

## 6.2 Brand Alignment Intro

### 목적

데이터 분석이 브랜드 이해에서 출발한다는 전제를 설명한다.

### 카피 구조

헤드라인:

```text
분석은 브랜드의 방향을
이해하는 것에서 시작합니다
```

강조 문장:

```text
소비자 데이터를 읽기 전에, 브랜드가 향하는 기준을 먼저 맞춥니다.
```

설명:

```text
성장 목표와 보유 자산, 캠페인 맥락을 정리해야 소비자 데이터가 단순한 리포트가 아니라
이번 OOH 캠페인이 해결해야 할 질문으로 이어집니다.
```

### 그리드

PC:

```css
.alignment-grid {
  grid-template-columns: repeat(3, minmax(0, 1fr));
  gap: 18px;
  max-width: 1040px;
}
```

모바일:

```css
grid-template-columns: 1fr;
```

### 카드 규칙

- 카드 배경: `#f6f6f6`
- 테두리 없음
- 라운드: `8px`
- 이미지 비율: `16 / 9`
- 카드 내부 텍스트는 PC 중앙 정렬, 모바일 좌측 정렬

카드 항목:

1. 성장 목표
2. 브랜드 자산
3. 캠페인 맥락

## 6.3 Problem / Solution Matrix

### 목적

브랜드 상황에 따라 먼저 봐야 할 데이터와 판단이 달라진다는 것을 시각적으로 보여준다.

### 카피 구조

헤드라인:

```text
OOH 캠페인의 방향을 더 선명하게 만들고
싶으시죠?
```

서브카피:

```text
브랜드가 처한 상황에 따라 먼저 봐야 할 데이터와 도출해야 할 판단은 달라집니다.
애드타입은 목표, 데이터, 산출물을 연결해 이번 캠페인이 먼저 답해야 할 질문을 정리합니다.
```

### 매트릭스 구조

행:

1. 분석 초점
2. 캠페인 상황
3. 활용 데이터
4. 도출 결과

열:

1. 브랜드 이해
2. 소비자 맥락
3. 브랜드 위치
4. 구매 이유
5. 과제·KPI

### 디자인 규칙

- 매트릭스 배경은 `#151515`
- 선택 상태는 `#EB4200`
- 행 라벨은 중앙 또는 좌측에 명확히 고정한다.
- PC에서는 전체 매트릭스를 보여준다.
- 모바일에서는 전체 표를 억지로 축소하지 않는다.
- 모바일은 체중계 게이지처럼 현재 집중 키워드가 크게 보이고, 좌우 키워드는 반쯤 걸리며 흐림 처리되는 구조를 사용한다.

### CTA

강조 CTA:

```css
background: #EB4200;
color: #fff;
hover background: #151515;
```

버튼 카피 예:

```text
분석 상담하기
```

## 6.4 Product / System Detail

### 목적

기술 모듈을 설명하되, 단순 기능이 아니라 서비스 품질을 높이는 운영체계로 보여준다.

### 기본 구성

각 시스템 카드는 다음 구조를 따른다.

1. 인덱스
2. 분석 관점
3. 시스템명
4. 헤드라인
5. 설명 본문
6. 기능 불릿
7. 캠페인 판단
8. 영상 또는 시원한 화면 캡처
9. 짧은 캡션

### PC 그리드

```css
.product-showcase-item {
  display: grid;
  grid-template-columns: minmax(0, .82fr) minmax(0, 1.18fr);
  gap: 48px;
  min-height: 520px;
  padding: 52px 44px;
  background: #f6f6f6;
  border: 0;
  border-radius: 8px;
}
```

교차 레이아웃:

- 1번째 카드: 텍스트 왼쪽, 영상 오른쪽
- 2번째 카드: 영상 왼쪽, 텍스트 오른쪽
- 3번째 카드: 텍스트 왼쪽, 영상 오른쪽

### 모바일 구조

모바일에서는 가독성을 위해 순서를 조정한다.

1. 헤드라인과 설명
2. 영상
3. 기능 불릿
4. 캠페인 판단

```css
.product-showcase-copy {
  display: contents;
}
```

### 카드 간격

- PC 카드 간격: `48px`
- 모바일 카드 간격: `34px`

### 비주얼 규칙

- 화면은 작게 겹치지 않는다.
- 실제 SurveyRadar, ReviewRadar, Mental Market 화면은 넓고 시원하게 보여준다.
- 여러 화면을 콜라주처럼 겹치는 방식은 조잡해 보일 수 있으므로 피한다.
- 영상/화면 위에 텍스트를 많이 얹지 않는다.
- 캡션은 영상 아래에 짧게 둔다.

### 현재 시스템 예시

#### Mental Market

관점: 소비자 맥락  
역할: 브랜드가 떠올라야 할 상황을 찾는다.

캠페인 판단:

```text
OOH에서 반복 노출할 상황, 관심사, 메시지 기회
```

#### SurveyRadar

관점: 브랜드 위치  
역할: 브랜드의 현재 위치를 데이터로 진단한다.

캠페인 판단:

```text
이번 캠페인이 개선해야 할 브랜드 지표와 타깃
```

#### ReviewRadar

관점: 구매 언어  
역할: 구매자의 실제 언어에서 선택 이유를 찾는다.

캠페인 판단:

```text
카피와 비주얼로 전환할 구매 이유와 장벽
```

### 결론 문장

PC에서는 박스를 적용하지 않고 텍스트만 둔다.

```text
브랜드의 성장 목표와 소비자의 실제 신호를 연결해,
OOH 캠페인이 누구에게 어떤 인식을 남겨야 하는지 정의합니다.
```

모바일에서는 `#f6f6f6` 박스를 유지해 읽는 덩어리를 만든다.

## 6.5 FAQ

### 목적

독자가 페이지를 읽고 남길 수 있는 실무적 질문에 답한다.

### 디자인

```css
.faq-next-section {
  background: #f9f9f9;
  padding: 112px 0 82px;
}

.faq-wrap {
  grid-template-columns: minmax(260px, .82fr) minmax(0, 1.45fr);
  gap: 86px;
}

.faq-title {
  font-weight: 540;
}

.faq-question {
  font-weight: 540;
}

.faq-answer {
  font-weight: 400;
}
```

### 규칙

- FAQ 섹션은 다른 회색 섹션보다 더 밝게 둔다.
- 큰 박스나 카드로 감싸지 않는다.
- 질문 목록은 얇은 기능적 구분선만 사용한다.
- 섹션 구분용 장식선은 사용하지 않는다.
- FAQ 제목과 질문은 `540`으로 통일한다.
- FAQ는 중앙 정렬 대형 헤드라인 섹션이 아니라, 좌측 설명과 우측 질문 리스트의 마무리 모듈로 구성한다.
- FAQ와 Final CTA는 별도 섹션처럼 분리하지 않고 같은 `#f9f9f9` 배경 안에서 하나의 흐름으로 이어지게 한다.

## 6.6 Final CTA

### 목적

페이지의 결론을 명확한 다음 행동으로 연결한다.

### 디자인

```css
.next-step-cta {
  display: grid;
  grid-template-columns: 158px minmax(0, 1fr) auto;
  gap: 46px;
  min-height: 300px;
  padding: 82px 64px;
  background: #151515;
  color: #fff;
  border-radius: 8px;
}

.next-step-copy h2 {
  font-weight: 540;
}

.next-step-btn {
  font-weight: 560;
}
```

### 규칙

- 마지막 CTA는 얇은 띠처럼 보이면 안 된다.
- 충분한 높이를 확보해 페이지의 강한 결말처럼 보이게 한다.
- 상단 주황색 라인 같은 장식 요소는 사용하지 않는다.
- CTA 아래쪽 여백은 과하게 남기지 않는다.
- CTA 제목은 `540`, CTA 버튼은 `560`을 기준으로 한다.
- CTA는 `다음 단계 / 단계 번호 / 다음 페이지명 / 설명 / 액션 버튼` 구조를 유지한다.
- 시스템 상세 페이지 간에는 CTA 문법을 새로 만들지 않는다.

### 버튼 시스템

Primary:

```css
background: #EB4200;
border: 1px solid #EB4200;
color: #fff;
```

Primary hover:

```css
background: #050505 또는 #151515;
```

Secondary on dark:

```css
background: transparent;
border: 1px solid rgba(255,255,255,.36);
color: #fff;
```

Secondary hover:

```css
background: #fff;
color: #151515;
```

## 7. 이미지/영상 사용 원칙

### 7.1 Hero

- 실제 서비스 화면, OOH 현장, 분석 맥락이 느껴지는 영상이 우선이다.
- 어둡게 딤 처리해 텍스트를 살린다.
- 배경은 분위기용이 아니라 페이지 주제를 암시해야 한다.

### 7.2 카드 이미지

- 브랜드 이해 카드에는 추상 일러스트보다 실제 업무 장면 이미지를 사용한다.
- 단, 너무 스톡처럼 보이지 않도록 실제 기획, 분석, 회의, 매체 검토 맥락이 드러나는 이미지가 좋다.

### 7.3 시스템 상세 영상

- 제품 화면은 시원하게 보여준다.
- 작은 썸네일 여러 개보다 대표 영상/화면 하나가 더 설득력 있다.
- 텍스트를 영상 위에 덮지 않는다.

## 8. 모바일 원칙

### 8.1 Hero

- Hero는 모바일에서 너무 복잡하면 안 된다.
- `01.`은 타이틀 위에 작게 분리한다.
- 헤드라인은 가능하면 한 줄에 들어오게 조정한다.
- 서브카피는 PC보다 살짝 덜 강조해 배경과 경쟁하지 않게 한다.

### 8.2 Matrix

- 전체 표를 축소해서 보여주지 않는다.
- 한 번에 하나의 핵심 키워드를 크게 보여준다.
- 좌우 키워드는 반쯤 걸치게 보여 “다른 선택지가 있다”는 느낌만 준다.

### 8.3 Product/System 카드

모바일 순서:

1. 텍스트
2. 영상
3. 불릿
4. 캠페인 판단

텍스트 다음에 영상을 넣어 읽는 흐름을 한 번 끊어준다.  
그 다음 기능 불릿을 배치하면 정보가 더 잘 들어온다.

### 8.4 결론 문장

- PC: 텍스트만
- 모바일: `#f6f6f6` 박스 유지

## 9. 금지 규칙

다음 요소는 Adtype 시스템 페이지에서 피한다.

- 섹션 구분용 장식선
- 의미 없는 영어 라벨
- 파란색 강조 텍스트
- 과한 볼드
- 카드 안에 또 카드가 들어가는 구조
- 화면 캡처 여러 개를 겹쳐 조잡하게 만드는 구성
- 너무 마일드한 베이지/주황빛 회색 배경
- 얇은 띠처럼 약한 마지막 CTA
- 제품 기능만 나열하고 캠페인 판단으로 연결하지 않는 설명

```text

- 이 시스템이 OOH 캠페인에서 해결하는 핵심 역할

- 3개 기준 카드

Problem / Solution Matrix
- 실무자가 느끼는 문제
- 상황별로 달라지는 판단 기준
- 목표, 데이터, 산출물 연결

Product / System Detail
- 시스템 1
- 시스템 2
- 시스템 3
- 각 시스템마다 캠페인 판단을 명확히 제시

FAQ

Final CTA
- 다음 단계 페이지 또는 상담으로 연결
```

## 11. 핵심 문장 원칙

Adtype 시스템 페이지의 카피는 아래 문장 구조를 따른다.

```text
우리는 무엇을 대신합니다
```

보다는

```text
우리는 어떤 데이터를 연결해, 캠페인이 판단해야 할 질문을 정리합니다
```

에 가깝게 쓴다.

좋은 방향:

- 소비자 데이터를 읽기 전에 브랜드가 향하는 기준을 먼저 맞춥니다.
- 브랜드가 처한 상황에 따라 먼저 봐야 할 데이터와 도출해야 할 판단은 달라집니다.
- 검색, 설문, 리뷰 데이터를 연결해 캠페인의 과제와 메시지 기회를 찾습니다.
- OOH 캠페인이 누구에게 어떤 인식을 남겨야 하는지 정의합니다.

피해야 할 방향:

- 전략을 대신 세워드립니다.
- 모든 문제를 해결합니다.
- AI가 자동으로 최적화합니다.
- 압도적인 기술력으로 캠페인을 혁신합니다.

## 12. 최종 감각

Adtype의 기술/시스템 페이지는 다음 느낌을 목표로 한다.

- 차갑게 정리되어 있지만 딱딱하지 않다.
- 기술적이지만 제품 판매 페이지처럼 보이지 않는다.
- 광고 서비스 회사이지만 감이 아니라 시스템으로 움직이는 조직처럼 보인다.
- 화면과 영상은 시원하고, 텍스트는 선명하며, 장식은 최소화한다.
- 오렌지는 적게 쓰되, 쓸 때는 확실하게 쓴다.
