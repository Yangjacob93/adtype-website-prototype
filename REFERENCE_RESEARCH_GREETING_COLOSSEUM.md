## 2. 그리팅 IA 분석

그리팅은 크게 `제품 기능`과 `솔루션`을 분리한다. 이 점이 중요하다.

### IA 구조

상단 네비게이션은 다음 축으로 구성된다.

- 왜 그리팅인가
- 제품
  - 모집: 채용 홈페이지, 인재풀 구축, 다이렉트 소싱
  - 관리: 공고 관리, 지원자 관리, 평가 관리
  - 운영: 면접 일정 조율, 지원자 연락, 채용 데이터 분석
- 솔루션
  - 기업 규모: 중견 · 대기업, 중소기업, 스타트업
  - 산업: IT · 기술, 제조 · 생산, 유통 · 커머스, 의료 · 제약, 패션 · 뷰티
- 고객 사례
- 가격
- 유용한 자료
- 도입 문의

핵심은 “제품 기능”은 기능 단위로 깊게 설명하고, “솔루션”은 고객 유형별로 어떤 기능 조합이 필요한지를 빠르게 보여준다는 점이다.

### 그리팅 솔루션 페이지의 공통 구조

기업 유형별/산업별 솔루션 페이지는 대체로 아래 구조를 따른다.

1. 히어로
   - 작은 라벨: 스타트업, 중견 · 대기업, IT · 기술 등
   - H1: 해당 고객군의 핵심 과제를 한 문장으로 압축
   - 서브카피: 그리팅이 후보자 발굴부터 입사까지 돕는다는 설명
   - CTA: 무료 체험하기, 도입 문의하기
   - 고객 로고 또는 관련 브랜드 로고

2. 신뢰 수치
   - 연간 내부 인재풀 증가율
   - 채용 리드타임 단축
   - 생산성 향상
   - 지원자 관리 수 등

3. 고객 사례
   - 3개 카드
   - 고객사/팀명/인터뷰 요약
   - 실사 기반 이미지 또는 인터뷰 썸네일
   - `인터뷰 확인하기` CTA

4. 관련 기능 묶음
   - 해당 고객군이 필요로 하는 기능 3~4개
   - 예: 원티드 연동, 채용 업무 자동화, 아웃바운드 채용
   - 페이지마다 기능 조합이 다르다.

5. 고객 지원
   - 가이드북, 커뮤니티, 서비스 소개서, 고객 사례, 컨설팅 등

6. FAQ

7. 하단 CTA

### 그리팅 기능 상세 페이지 구조

`채용 데이터 분석` 기능 상세 페이지는 솔루션 페이지보다 더 깊다.

1. 히어로
   - 작은 도움말 링크: `채용 데이터 분석 기반으로 더 나은 의사결정을 하고 싶다면? 가이드북`
   - H1: `채용 데이터가 한곳에 / 인사이트가 한눈에`
   - 서브카피: `한눈에 펼쳐진 채용 데이터를 기반으로, 모집부터 선발까지 채용 성과를 최적화하세요.`
   - CTA: 무료 체험하기, 도입 문의하기
   - 대형 제품 UI 이미지

2. Why 섹션
   - 라벨: 채용 현황 분석
   - H2: `핵심 인재 확보에 / 채용 데이터가 필요한 이유`
   - 2열 카드
   - 각 카드: 작은 라벨 + 타이틀 + 설명 + 제품 UI 조각

3. 세부 기능 섹션
   - 채용 대시보드
   - 채용 성과 보고서
   - 링크 공유/엑셀 추출 등 기능 단위

4. 고객사 인터뷰

5. FAQ

### 그리팅 디자인 시스템

- 폰트: Pretendard 계열
- 전체 톤: 밝은 화이트/라이트 블루, 신뢰감 중심
- 키 컬러: 블루
- 히어로:
  - 솔루션 페이지는 그라데이션 배경 + 중앙 정렬 카피
  - 기능 상세는 화이트 배경 + 대형 UI 이미지
- H1:
  - 큰 편이지만 과도하게 장식적이지 않음
  - 2줄 내외로 명확한 효익을 말함
- 카드:
  - 라운드가 큰 편
  - 배경은 `#F7F8FA`에 가까운 아주 연한 회색
  - 그림자는 약함
- 시각자료:
  - 제품 UI 영상/이미지 적극 사용
  - 고객 사례는 실사 인터뷰 카드
  - 로고/수치/인터뷰로 신뢰를 빠르게 쌓음

### 그리팅에서 배울 점

애드타입에 바로 적용할 점은 다음이다.

- 상세 페이지에는 “기능 설명”보다 “이 과제의 고객이 왜 우리를 써야 하는가”를 먼저 보여줘야 한다.
- 각 상세 페이지의 H2 묶음은 페이지별로 달라야 한다.
  - 지금 애드타입 1~6번 상세 페이지가 너무 공통 템플릿처럼 보이면 설득력이 떨어진다.
- 고객 사례가 부족해도 “수치/상황/과제/방식”으로 신뢰 블록을 만들 수 있다.
- 메인 솔루션 페이지는 얕아도 괜찮지만, 기능 상세 페이지는 깊어야 한다.
- 제품 UI나 시스템 화면은 단순 장식이 아니라 “이 회사가 실제로 일하는 방식”의 증거로 배치해야 한다.

   - 맞춤형 디지털 솔루션

  - 거의 모든 서비스 페이지가 실사 이미지 풀블리드
  - 어두운 딤 위에 흰색 H1
  - 서비스가 실제 운영 현장과 연결되어 있다는 느낌을 줌
- H1:
  - 매우 큼
  - 서비스명을 직관적으로 보여줌
- H2:
  - 40px 내외, 600 weight 수준
  - 제품/서비스의 기능성을 직접 말함
- 그리드:
  - 센터 정렬 섹션
  - 프로세스 바/매트릭스/아이콘 카드
  - 실사 이미지 + 설명 조합
- 시각자료:
  - 실사 창고/물류센터/작업자 이미지
  - 서비스 범위 다이어그램
  - IT 솔루션 페이지에서는 제품 UI 이미지

### 콜로세움에서 배울 점

- 복잡한 서비스를 설명할 때는 “우리가 뭘 해드립니다”보다 “어느 과정까지 커버합니다”가 강하다.
- 애드타입도 OOH 캠페인 상세 페이지에서 다음을 더 명확히 보여줘야 한다.
  - 진단
  - 캠페인 과제화
  - 공간/매체 설계
  - 크리에이티브/소재
  - 집행
  - 성과 분석
  - 다음 캠페인 루프
- 콜로세움처럼 프로세스 바나 범위 매트릭스를 쓰면, 애드타입의 “에이전시+기술+운영” 범위를 더 잘 이해시킬 수 있다.
- 실사 이미지는 신뢰를 만든다. 애드타입도 매체 현장 사진, 시스템 화면, 캠페인 결과물을 더 적극적으로 섞는 편이 좋다.

## 4. 두 사이트의 차이

| 항목 | 그리팅 | 콜로세움 |
|---|---|---|
| IA 기준 | 제품 기능 + 고객 유형/산업 | 서비스 유형/운영 문제 |
| 솔루션 페이지 깊이 | 비교적 짧고 신뢰/사례 중심 | 길고 범위/프로세스 중심 |
| 기능 상세 | 별도 feature 페이지에서 깊게 설명 | 서비스 페이지 안에서 기능까지 설명 |
| 히어로 | 그라데이션 또는 화이트, 중앙 정렬 | 실사 풀블리드, 딤, 대형 H1 |
| 신뢰 장치 | 고객 로고, 수치, 인터뷰 카드 | 현장 실사, 프로세스 범위, 운영 항목 |
| 카피 톤 | 쉽고 친절함, 효익 중심 | 기능적이고 직접적, 서비스 범위 중심 |
| 시각자료 | 제품 UI, 인터뷰, 로고 | 현장 이미지, 다이어그램, 시스템 UI |

## 5. 애드타입에 적용할 시사점

현재 애드타입 상세 페이지에서 “내용이 없어졌다”는 느낌이 든 이유는 다음으로 보인다.

1. 각 상세 페이지가 공통 템플릿 위주로 정리되면서, 페이지별 고유한 문제/상황/실행 범위가 약해졌다.
2. 브랜드마케터/BM이 실제로 보는 질문이 부족했다.
3. “데이터 기반으로 한다”는 말은 있으나, 어떤 데이터가 어떤 판단으로 이어지는지 단계가 부족했다.
4. 캠페인 결과물/현장/시스템 이미지가 충분히 근거처럼 작동하지 않았다.
5. 페이지별 CTA 이전에 “그래서 애드타입이 어디까지 해주는가”가 더 촘촘히 보여야 한다.

### 애드타입 상세 페이지 권장 구조

그리팅과 콜로세움을 섞으면 애드타입 상세 페이지는 아래 구조가 가장 적합하다.

1. 히어로
   - 페이지별 과제명
   - 브랜드마케터가 실제로 쓰는 언어의 H1
   - 짧은 서브카피
   - CTA
   - 관련 실사/시스템/캠페인 이미지

2. 이 과제가 생기는 상황
   - 브랜드사의 실제 고민 3~4개
   - 예: 인지도는 낮다, 특정 생활권에서 존재감이 약하다, 경쟁 브랜드 대비 덜 보인다

3. 애드타입의 과제 전환
   - 브랜드 과제 → 데이터 질문 → OOH 캠페인 과제
   - 그리팅처럼 쉬운 문장으로, 콜로세움처럼 범위를 명확히

4. 데이터 기반 진단
   - Mental Market
   - Survey/Review Radar
   - Adtype Map/Simulator
   - MMS
   - Performance System
   - 페이지별로 필요한 도구만 선택

5. 캠페인 설계 범위
   - 타겟
   - 생활권
   - 동선
   - 매체
   - 소재/크리에이티브
   - 집행/성과
   - 콜로세움식 프로세스 바 또는 매트릭스 추천

6. 세부 전략 유형
   - 기존의 3개 카드/모달은 유지 가능
   - 단, 카드마다 “상황 → 전략 → 확인 지표”를 넣어야 함

7. 시각 근거
   - 시스템 화면
   - 매체 현장 이미지
   - 캠페인 결과물
   - 수치/지표

8. 성과 관리
   - 노출/도달/빈도
   - 검색량/브랜드 지표
   - 지역별 반응
   - 다음 캠페인 제안

9. FAQ

10. CTA

### 페이지별로 더 채워야 할 내용 예시

#### 1. 브랜드 존재감 확대

- 상황:
  - 브랜드를 아직 충분히 모르는 타겟에게 반복적으로 발견되어야 한다.
  - 특정 지역/생활권에서 브랜드 존재감을 먼저 만들어야 한다.
  - 경쟁 브랜드 대비 “자주 보이는 브랜드”가 되어야 한다.
- 데이터:
  - 타겟 생활권
  - 이동 동선
  - 체류 공간
  - 검색량 변화
  - 지역별 반응
- 캠페인 과제화:
  - 인지도 확대 → 타겟 생활권 내 반복 접점 설계
- 세부 전략:
  - 브랜드 인지도 형성
  - 브랜드 대세감 형성
  - 카테고리 존재감 확보
- 필요한 시각자료:
  - 지도/동선 시스템 화면
  - 주요 매체 조합 예시
  - 실제 캠페인 실사

#### 2. 브랜드 포지셔닝 강화

- 상황:
  - 브랜드는 알려져 있지만 어떤 이미지로 기억되는지 흐릿하다.
  - 카테고리 내 경쟁 브랜드와 의미가 겹친다.
  - 프리미엄/신뢰/전문성/취향 같은 이미지를 강화해야 한다.
- 데이터:
  - 검색 키워드 클러스터
  - 리뷰/서베이
  - 경쟁 브랜드 기억 구조
  - 소비자 언어
- 캠페인 과제화:
  - 포지셔닝 강화 → 브랜드가 공략할 기억 구조와 공간 맥락 설계
- 필요한 시각자료:
  - Mental Market 화면
  - 키워드 클러스터
  - 브랜드/경쟁사 인식 매트릭스

#### 3. 구매 이유 형성

- 상황:
  - 인지도는 있으나 구매 고려로 이어지지 않는다.
  - 제품 장점이 소비자 언어로 정리되지 않았다.
  - 리뷰/VOC 안의 강점을 캠페인 메시지로 전환하지 못했다.
- 데이터:
  - 리뷰 텍스트
  - 서베이
  - 검색 맥락
  - 구매 장벽/구매 동기
- 캠페인 과제화:
  - 구매 고려도 개선 → 구매 이유를 소비자 언어로 정리하고 고려 순간의 접점에 배치
- 필요한 시각자료:
  - Survey/Review Radar
  - 메시지 후보 카드
  - 구매 접점 주변 매체 예시

#### 4. 신제품·리뉴얼 인식 전환

- 상황:
  - 신제품 차이가 빠르게 이해되어야 한다.
  - 리뉴얼 포인트가 소비자에게 전달되지 않는다.
  - 출시 초기 검색량/체험/방문/구매 고려 신호가 필요하다.
- 데이터:
  - 초기 타겟
  - 출시 집중 생활권
  - 제품 차별점
  - 검색/인식 변화
- 캠페인 과제화:
  - 신제품 홍보 → 차별점의 초기 인식 전환 캠페인
- 필요한 시각자료:
  - 런칭 타임라인
  - 매체별 소재 베리에이션
  - 캠페인 현장 이미지

#### 5. 오프라인 접점 기반 유입 확대

- 상황:
  - OOH가 검색/방문/앱/매장/리테일 행동으로 이어져야 한다.
  - 특정 상권이나 매장 주변에서 유입을 만들어야 한다.
- 데이터:
  - 구매 접점
  - 생활권
  - 이동 동선
  - 검색/방문/쿠폰/유입 신호
- 캠페인 과제화:
  - 유입 확대 → 생활권과 구매 접점을 연결하는 오프라인 행동 설계
- 필요한 시각자료:
  - 미디어 플래닝 시스템
  - 상권 지도
  - 퍼널/동선 다이어그램

#### 6. 브랜드 지표 개선과 캠페인 고도화

- 상황:
  - 캠페인 이후 브랜드 지표가 실제로 움직였는지 설명해야 한다.
  - 다음 캠페인의 타겟/지역/메시지/매체 조합을 개선해야 한다.
- 데이터:
  - 노출/도달/빈도
  - 검색량
  - 상기도/고려도
  - 지역별 반응
  - 캠페인 전후 서베이
- 캠페인 과제화:
  - 성과 분석 → 다음 캠페인 개선 루프
- 필요한 시각자료:
  - 성과 대시보드
  - 변화 그래프
  - 다음 전략 제안 보드

## 6. 애드타입 상세 페이지 디자인 방향 제안

### 유지할 것

- 토스식의 넓은 여백
- 과하지 않은 폰트 웨이트
- 화이트/라이트 그레이 중심
- 키 컬러 `#EB4200`
- 직관적 H1
- 페이지별 전용 시각자료

### 보강할 것

- 각 페이지별 “현실적 문제 상황” 섹션
- 고객이 실제로 하는 질문
- 데이터가 전략으로 변환되는 과정
- 프로세스/범위 매트릭스
- 실사 이미지와 시스템 화면의 혼합
- FAQ
- 페이지별 차별화된 하위 전략 카드

### 피해야 할 것

- 모든 페이지가 같은 레이아웃/카드/문장 구조를 반복하는 것
- 추상적인 “데이터 기반” 반복
- 너무 AI/SaaS 대시보드처럼 보이는 시각자료만 쓰는 것
- H1/H2에 과도한 700~800 웨이트를 쓰는 것
- 제품명 중심으로만 말하고 브랜드마케터의 과제를 놓치는 것

## 7. 다음 작업 제안

이 리서치를 기준으로 다음 단계는 `1. 브랜드 존재감 확대` 상세 페이지를 다시 설계하는 것이다.

권장 순서:

1. 현재 페이지의 섹션을 전부 나열한다.
2. 그리팅/콜로세움 레퍼런스 기준으로 부족한 섹션을 표시한다.
3. `브랜드 존재감 확대` 페이지 전용 IA를 다시 만든다.
4. 카피를 페이지 전용으로 재작성한다.
5. 시각자료를 시스템/실사/다이어그램으로 재배치한다.
6. 동일한 디자인 시스템 안에서 다른 상세 페이지와 구분되는 레이아웃을 적용한다.

첫 번째 페이지에서 확정한 구조를 나머지 5개 페이지에 그대로 복사하지 말고, 페이지별로 “같은 브랜드, 다른 문제를 해결하는 페이지”처럼 변주해야 한다.
